<objective>
## Goal
Build a dedicated `/analytics/aram` page that shows ARAM-specific performance stats — win rate by champion, KDA averages, damage share, multi-kill highlights, and a per-champion breakdown table — computed from cached match data in Supabase.

## Purpose
ARAM is a distinct game mode with different performance signals than SR. Players who play ARAM frequently (like Bjarke) need a dedicated view that filters out SR noise and surfaces ARAM-specific insights the desktop version already provides.

## Output
- `GET /api/analytics/aram` route returning aggregated ARAM stats for the authed user
- `/analytics/aram` page with 5 stat sections (overview, champion breakdown, highlights, damage stats, game length)
- 3 reusable ARAM-specific components
</objective>

<task type="auto">
  <name>Task 1: ARAM analytics API route</name>
  <files>src/app/api/analytics/aram/route.ts</files>
  <action>
    Check if the route exists from Phase 3. If it exists, read it and enhance to return the full payload below. If not, create it.

    Route: GET /api/analytics/aram
    Auth: validate Supabase session — return 401 if no session
    Query: join matches + match_stats tables where game_mode = 'ARAM' and puuid = user's linked puuid

    Compute and return:
    - total_aram_games: count of ARAM matches
    - win_rate: wins / total (0–100, rounded to 1 decimal)
    - avg_kda: average (kills + assists) / max(deaths, 1) across all ARAM games
    - avg_damage_share: average damage_dealt_to_champions / team_total_damage (from full_stats JSONB)
    - avg_game_length_minutes: average game_duration from raw_data JSONB
    - multi_kill_counts: { triple: N, quadra: N, penta: N } summed from full_stats JSONB largestMultiKill field
    - champion_breakdown: array of { champion_id, champion_name, games, wins, avg_kda, avg_damage_share } sorted by games desc, max 20 entries

    Extract damage/multikill data from full_stats JSONB using Postgres json operators.
    Use a single SQL query with aggregation rather than fetching all rows to JS — avoids N+1 and handles large match histories.

    Avoid: exposing raw_data to the client, any Riot API calls (use only cached Supabase data)
  </action>
  <verify>curl -H "Cookie: ..." http://localhost:3000/api/analytics/aram returns JSON with all 7 fields and champion_breakdown array</verify>
  <done>AC-1 satisfied: API returns full ARAM stats payload from Supabase cache</done>
