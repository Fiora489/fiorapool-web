<objective>
Ship the server-rendered visual build card (edge `ImageResponse`) that the Creator UI will download + the pure `build-diff` computation module the Hub UI will render. No viewer UI — both consumed from UI phases later.
</objective>

<scope>
**IN:**
- `/api/share/build-card?buildId=...&layout=hero|compact|scoreboard` — edge runtime, `next/og` `ImageResponse`. Renders:
  - Hero: champion splash + build name + roles + full items grid + runes badge + core stats panel
  - Compact: items grid only + champion icon + name (for Discord/Twitter thumbnails)
  - Scoreboard: league-client-style panel — items in order, runes as icons, spells, stat totals
- `card-layout.ts` — pure layout helpers + shared style tokens (no runtime JSX; returns layout-descriptor objects the route renders)
- `build-diff.ts` — pure: `computeBuildDiff(a, b): BuildDiff` returning:
  - items: { added[], removed[], reordered[] } per block
  - runes: { primaryStyleChanged, keystoneChanged, minorsDiff[], shardsDiff[] }
  - spells: { spellsChanged: boolean, pair: [a, b] }
  - skills: { orderChanged: boolean, maxPriorityChanged: boolean }
  - meta: { tagsAdded[], tagsRemoved[], descriptionChanged: boolean }
- Auth: build-card route requires signed-in owner for private builds; public builds readable by anyone
- Rate limit: share/build-card route limited to 60 req / user / hour (reuses existing ratelimit util in `src/lib/ratelimit.ts` if present; else inline simple token bucket)

**OUT (→ UI phases):**
- Download-PNG button on the editor (Phase 66)
- Build diff page (`/builds/diff?a=...&b=...`) — Phase 67
- Social meta tags on build detail pages using this endpoint (Phase 67)
</scope>
