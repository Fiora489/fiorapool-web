  <action>
    Create the 10 tables listed in MILESTONE-v1.3-BUILD-CREATOR.md under "Data Model Sketch".

    Concrete columns per table (all tables get `id uuid pk default gen_random_uuid()`, `created_at timestamptz default now()`, `updated_at timestamptz default now()` except join tables which use composite PKs):

    - custom_builds:
        user_id uuid not null references auth.users(id) on delete cascade,
        champion_id text not null,
        name text not null,
        description_md text default '',
        roles text[] not null default '{}',
        build_tags text[] not null default '{}',
        patch_tag text not null,
        last_validated_patch text,
        combos text[] not null default '{}',
        max_priority text,
        warding_note text,
        trinket text,
        spell1 text, spell2 text, spell_alt_note text,
        rune_page_id uuid references custom_rune_pages(id) on delete set null,
        skill_order smallint[18],
        is_public boolean not null default false,
        opt_in_aggregate boolean not null default false
    - custom_build_blocks:
        build_id uuid references custom_builds(id) on delete cascade,
        block_type text check (block_type in ('starting','early','core','situational','full','boots')),
        position smallint not null,
        items jsonb not null default '[]',
        power_spikes smallint[] default '{}',
        gold_total integer default 0,
        primary key (build_id, block_type)
    - custom_matchup_notes:
        build_id uuid references custom_builds(id) on delete cascade,
        enemy_champion_id text,
        difficulty text check (difficulty in ('easy','even','hard','counter')),
        note text default '',
        threats jsonb default '[]',
        primary key (build_id, enemy_champion_id)
    - custom_item_swaps:
        id uuid pk,
        build_id uuid references custom_builds(id) on delete cascade,
        condition_text text not null,
        from_item integer not null,
        to_item integer not null,
        position smallint default 0
    - custom_rune_pages:
        id uuid pk,
        user_id uuid not null references auth.users(id) on delete cascade,
        name text not null,
        primary_style smallint not null,
        keystone integer not null,
        primary_minors integer[3] not null,
        secondary_style smallint not null,
        secondary_minors integer[2] not null,
        shards smallint[3] not null
    - build_bookmarks:
        user_id uuid references auth.users(id) on delete cascade,
        build_id uuid references custom_builds(id) on delete cascade,
        created_at timestamptz default now(),
        primary key (user_id, build_id)
    - build_collections:
        id uuid pk,
        user_id uuid references auth.users(id) on delete cascade,
        name text not null,
        description text default ''
    - build_collection_items:
        collection_id uuid references build_collections(id) on delete cascade,
        build_id uuid references custom_builds(id) on delete cascade,
        position smallint not null,
        primary key (collection_id, build_id)
    - build_item_block_templates:
        id uuid pk,
        user_id uuid references auth.users(id) on delete cascade,
        champion_id text not null,
        block_type text not null,
        items jsonb not null default '[]'
    - build_match_tags:
        build_id uuid references custom_builds(id) on delete cascade,
        match_id text not null,
        user_id uuid references auth.users(id) on delete cascade,
        won boolean not null,
        detected_at timestamptz default now(),
        primary key (build_id, match_id)

    RLS policies:
    - Enable RLS on every table.
    - Owner policies: `auth.uid() = user_id` for all select/insert/update/delete on user-owned tables.
    - Public read on `custom_builds WHERE is_public = true` (policy name: `public_builds_read`).
    - Child-table public reads through `EXISTS (SELECT 1 FROM custom_builds WHERE id = {child}.build_id AND (is_public = true OR user_id = auth.uid()))`.
    - Bookmarks/collections: owner-only always.

    Indexes:
    - `custom_builds (user_id, updated_at desc)` for personal lists
    - `custom_builds (champion_id, is_public) where is_public = true` for Hub queries (Phase 58 benefits)
    - `custom_builds (patch_tag) where is_public = true` for patch filter
    - `build_match_tags (build_id)` for WR rollup
    - `build_match_tags (user_id, build_id)` for personal WR
  </action>
