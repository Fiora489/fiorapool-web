<objective>
Domain logic for runes, summoner spells, skill order, and combos. Build the rune-tree data layer from Data Dragon, validators for skill order + combos, and server actions for the Rune Page Library CRUD and for attaching runes/spells/skills to a build.
</objective>

<scope>
**IN:**
- `rune-tree.ts` — loads Data Dragon `runesReforged.json`, exports `getRuneTree(patch)` returning typed tree (5 paths × keystones + minor rows); `validateRunePage(page)` pure function
- `summoner-spells.ts` — loads Data Dragon summoner spells, exports `getSummonerSpells(patch)` + `validateSpellPair(a, b)` (enforces Flash legality, summoner-rift-legal only by default)
- `skill-order.ts` — validators for an 18-slot skill order: only Q/W/E/R allowed; R only at levels 6/11/16; no ability exceeds 5 points (R: 3 points); max-priority string parser (e.g., "Q > E > W")
- `combos.ts` — free-form string validators (length ≤ 60 chars each; max 10 combos)
- Extended types in `src/lib/types/builds.ts`: `RunePage`, `RunePath`, `RuneId`, `SpellId`, `SkillSlot`, `SkillOrder`, `MaxPriority`
- Server actions in `actions.ts`:
  - `createRunePage(name, page)`
  - `updateRunePage(id, name, page)`
  - `deleteRunePage(id)`
  - `setBuildRunePage(buildId, runePageId | null)`
  - `setBuildSpells(buildId, spell1, spell2, altNote)`
  - `setBuildSkillOrder(buildId, order, maxPriority)`
  - `setBuildCombos(buildId, combos[])`

**OUT (→ Phase 64 UI):**
- Rune page editor visual tree selector
- Summoner spell picker modal
- 18-column skill grid
- Combos list editor
- `/builds/runes` library route and CRUD screens
</scope>
