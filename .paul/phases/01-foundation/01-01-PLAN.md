<task type="auto">
  <name>Task 2: Add GET /api/summoner/[puuid] route</name>
  <files>src/app/api/summoner/[puuid]/route.ts</files>
  <action>
    Create `src/app/api/summoner/[puuid]/route.ts` with a GET handler that:
    1. Creates a Supabase server client (use existing pattern from `api/summoner/link/route.ts`)
    2. Calls `supabase.auth.getUser()` — returns 401 if no user
    3. Extracts `puuid` from route params (Next.js 16 App Router dynamic segment)
    4. Queries `summoner_profiles` where `puuid = params.puuid`
    5. Returns 404 if no row found, 200 with profile JSON if found

    RLS is already enforced at the DB level — any authenticated user can read any
    summoner_profiles row (RLS policy is `auth.uid() = user_id`, so users can only
    read their own). This is acceptable for Phase 1 since rival profiles aren't needed yet.

    Use the same import style as `api/summoner/link/route.ts` for consistency.
    Return type should be `{ profile: Tables<'summoner_profiles'> }` on success.

    IMPORTANT: Check how Next.js 16 App Router passes dynamic params to route handlers
    (the params object shape may have changed — read node_modules/next/dist/docs/ first).
  </action>
  <verify>
    curl -H "Cookie: ..." http://localhost:3000/api/summoner/{your-puuid}
    Should return 200 with profile JSON.
    Without auth cookie: should return 401.
    With unknown PUUID: should return 404.
    tsc --noEmit: no errors in this file.
  </verify>
  <done>AC-1 satisfied: GET /api/summoner/[puuid] returns correct status codes</done>
</task>
