<objective>
## Goal
Build dedicated `/progression/badges/mastery` page — per-champion mastery badges across 4 tiers (First Win / 10 Wins / 50 Wins / 100 Wins), filterable by champion name, earned status, and tier. Computed entirely from cached match data.

## Purpose
Adds a personal-progression dimension keyed to champion choices: the player can see at a glance which champions they've mastered and which are next.
</objective>

<acceptance_criteria>

## AC-1: computeChampionMastery returns full payload
```gherkin
Given matches with champion_name
When computeChampionMastery(matches) runs
Then output is { champions: ChampionMasteryRow[], totals: { championsPlayed, badgesEarned, totalBadges } }
And each ChampionMasteryRow has { name, games, wins, tiers: { tier, threshold, name, earned }[], topTier }
```

## AC-2: Page Renders Overview + Filters + Grid
```gherkin

When /progression/badges/mastery loads

```

```gherkin

Then the visible champion cards update to match
```

## AC-4: Mobile Functional

```gherkin
Then filter controls stack, card grid is single-column
```
