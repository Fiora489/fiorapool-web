<objective>
## Goal
Build dedicated `/progression/badges/chains` page showing all 10 chain badges (victory / streak / kda / cs / kills / xp / aram / veteran / consistent / pool) with per-chain tier progress, earned dates, and aggregate completion stats.

## Purpose
The Badge Chains feature in the desktop app is a 60-badge progression system across 10 themed chains. The web needs a dedicated page that surfaces this clearly — overview + per-chain cards with all tiers visible.
</objective>

<acceptance_criteria>

## AC-1: Overview Stats
```gherkin
Given a user with badges earned
When the page loads
Then the overview shows: total chains complete (X/10),
     total chain-badges earned (Y/60), next-up tier (closest unearned)
```

## AC-2: Per-Chain Cards
```gherkin
Given each of the 10 chains
When the page renders
Then each chain card shows: chain label, progress bar (earned/total),
     all tiers in order with icon + name + earned status,
     earned date for completed tiers
```

## AC-3: Mobile Functional
```gherkin
Given page at 390px
When loaded
Then chain cards stack vertically; tiers wrap as needed
```

## AC-4: Empty State
```gherkin
Given zero earned badges
When the page loads
Then overview shows 0/10 + 0/60 and chain cards still render with all tiers in unearned state
```

</acceptance_criteria>
