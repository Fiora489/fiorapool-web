<objective>
Wire the Build Creator into the existing XP + badge progression systems. Define XP events (create/publish/fork/first-to-patch/match-tagged-WR-threshold), register new badge chains, and ship badge evaluators that run on the relevant triggers. No UI — existing badge display surfaces pick these up automatically.
</objective>

<scope>
**IN:**
- `xp-events.ts` — event dispatchers the server actions call:
  - `onBuildCreated(userId, buildId)` → +50 XP
  - `onBuildPublished(userId, buildId)` → +100 XP
  - `onBuildForked(userId, originalBuildId)` → +25 XP to original creator (fork itself lands in later milestone; event stays dormant until then)
  - `onBuildBookmarked(userId, buildId, bookmarkerId)` → +10 XP to creator
  - `onBuildFirstToPatch(userId, buildId, patch)` → +200 XP (guarded: only first N hours of patch)
  - `onBuildMatchTagged(userId, buildId)` → +5 XP per match tagged (capped 5/day)
- Extends existing `src/lib/app-progress/xp.ts` with a union `XpEventType` additions; keeps event log row schema
- New badge chains (added to existing catalogue, not a separate system):
  - `Architect` — 1/10/50/100/250 builds created
  - `Publisher` — 1/5/25/100 builds made public
  - `Influencer` — 10/50/250 bookmarks on any single build
  - `Patch Responder` — 1/5/25 builds created within 24h of a patch drop
  - `Scholar` — 1/5/10 collections
  - `Archivist` — 1/5/10 rune pages saved
- `evaluators.ts` — 6 new badge evaluators that hook into the above events. Each pure-ish: given `(event, userContext) → BadgeAwardDelta[]`.
- Migration: inserts new badge rows into existing `badge_catalogue` (or equivalent) + any new columns needed.
- Server actions updated to emit events: `createBuild`, `setBuildPublic`, `stampLastValidated`, `bookmarkBuild`, `createRunePage`, `createCollection`.

**OUT (→ UI phases):**
- Surfacing new badges on the existing Badges pages (no changes needed — pages re-render automatically as catalogue grows)
- Level-up animations (already exist from v1.2)
</scope>
