<objective>
Ship the **domain logic** behind the item-composition editor so the UI phase (64) can render against a stable contract: Data-Dragon-backed items catalogue, pure stat + gold compute, autosave utility, extended `BuildMutationInput` types, and the `saveBuildDraft` server action. No UI in this phase.
</objective>

<scope>
**IN:**
- `items-catalogue.ts` — fetches item.json per patch via existing `src/lib/ddragon.ts`, caches via `unstable_cache` per patch+locale, exports `getItemsCatalogue(patch)` + `getItem(patch, id)` + `ItemRecord` type with id, name, iconUrl, gold.total, tags[], stats record, into[], from[], maps
- `stat-compute.ts` — `computeFinalStats(items)` pure function; returns `ComputedStats` struct (AD, AP, HP, armor, MR, haste, crit, MS, lifesteal, omnivamp). Additive across items; flat stats + percent stats handled separately. Documented simplifications (no unique-passive caps).
- `gold-compute.ts` — `computeBlockGold(items)` + `computeBuildGold(blocks)`
- `autosave.ts` — framework-agnostic debounce-save utility (state machine + timer) so the UI phase only needs to wire it. Pure; tested with fake timers.
- `BuildMutationInput` type extended with `blocks` map shape
- `saveBuildDraft(id, input)` server action — transactional update of `custom_builds` + replace of `custom_build_blocks`; computes and stores `gold_total` per block for cheap list rendering

**OUT (→ Phase 64, Build Creator UI):**
- Every React component: `BuildEditorShell`, `ChampionPicker`, `RoleMultiSelect`, `ItemBlockEditor`, `ItemPickerDialog`, `PowerSpikeMarker`, `LiveStatPanel`, `GoldSumBadge`, `AutosaveIndicator`
- Editor routes: `/builds/custom/new`, `/builds/custom/[id]/edit`
- Drag/drop integration, touch gestures, modals
</scope>
