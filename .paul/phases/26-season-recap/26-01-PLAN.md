<objective>
## Goal
Build new `/recap` page that summarises the user's "season" — total games, win rate, best champion, best role, most-played queue, longest streak, highlight games, days played, recent badges.

## Purpose
Players want a single shareable summary card showing what they accomplished. This page is the screen-friendly version; a future PNG export can use the same `lib/recap.ts` data.
</objective>

<acceptance_criteria>

## AC-1: computeRecap returns full payload
```gherkin
Given a user with cached matches and badges
When computeRecap(matches, badges, progress) runs
Then output includes: identity stats (totalGames, totalWins, winRate),
     daysPlayed, gamesPerDay, longestWinStreak, longestLossStreak,
     bestChampion ({name, games, winRate, avgKda}),
     bestRole ({role, games, winRate}),
     mostPlayedQueue ({queue, games}),
     highlights: { bestKda, mostKills, longestGame } each as {champion, value, captured_at, durationSeconds},
     recentBadges (last 5 earned with id, name, icon, earnedAt)
```

## AC-2: Page renders 5 sections
```gherkin
Given user visits /recap
When page loads
Then Header / Overview / Best (champion + role + queue) / Highlights / Recent Badges all render
```

## AC-3: Empty state
```gherkin
Given a user with 0 matches
When they visit /recap
Then a centered empty state shows ("No matches yet — play to populate your recap.")
```

## AC-4: Mobile functional
```gherkin
Given the page at 390px viewport
When loaded with data
Then all sections render without horizontal overflow
```

</acceptance_criteria>
