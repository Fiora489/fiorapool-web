<objective>
## Goal
Build dedicated `/progression/prestige/titles` page with 14 prestige titles, unlock criteria, current progress, and equip action that writes to `app_progress.prestige_title`.

## Purpose
Players want a tangible reward for crossing milestones — a chosen title to display. The schema already has `prestige_title` on app_progress; this page is the UI to choose one.
</objective>

<acceptance_criteria>

## AC-1: 14 titles defined with criteria + computed unlock status
```gherkin
Given matches + progress
When computePrestigeTitles(matches, progress) runs
Then output is { titles: TitleStatus[] (14), unlockedCount, equipped: string | null }
And each TitleStatus has { id, name, description, icon, currentValue, threshold, unlocked, progressLabel }
```

## AC-2: Page renders overview + grid + equip bar
```gherkin
Given user data
When /progression/prestige/titles loads
Then TitlesOverview, equipped-title bar, and TitleCard grid all render
```

## AC-3: Equip action persists
```gherkin
Given an unlocked title
When the user clicks Equip
Then a server action writes to app_progress.prestige_title
And the page revalidates to show the new equipped title
```

## AC-4: Mobile + Empty State
```gherkin
Given mobile 390px or no titles unlocked
Then layout works at 390px; locked titles still render with criterion shown
```

</acceptance_criteria>
