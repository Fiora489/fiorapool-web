<objective>
## Goal
Build dedicated `/progression/xp-multiplier` page that visualises the current XP multiplier (streak bonus), shows the per-streak-length XP table, and projects level progression curves under different streak scenarios.

## Purpose
The existing `xpForWin()` rewards streaks with up to +150 bonus XP per game (capped at 10-game streak). The web has no UI surfacing this — players don't see why they should keep streaks alive.
</objective>

<acceptance_criteria>

## AC-1: computeMultiplierStats returns streak + projection data
```gherkin
Given user matches and current app_progress (level, xp)
When computeMultiplierStats(matches, progress) runs
Then output includes:
- currentStreak (signed; positive = win streak, 0 if last was a loss)
- currentMultiplier (string like "+150 XP/game" at 10+ streak, "base" at 0)
- nextWinXp (XP value if next game is a win at current streak+1)
- streakTable: 11 rows (streak 0–10) with { streak, xpPerWin, bonus }
- projections: 3 scenarios (no streak / 5-streak / 10-streak) projecting next 10 games' XP and resulting level
```

## AC-2: Page Renders 3 Sections
```gherkin
Given user data

```

## AC-3: Mobile + Empty State
```gherkin

Then layout works at 390px; empty state for 0 matches surfaces baseline values (streak 0)
```
