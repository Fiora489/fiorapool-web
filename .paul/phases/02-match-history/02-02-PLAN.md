<objective>
## Goal
Add sync loading feedback via `useFormStatus` and a post-game summary strip showing the most recent match prominently above the match list.

## Purpose
Phase 2 ROADMAP scope includes "post-game summary component" and "mobile-first" UX. Currently the sync button gives no feedback (the native form just causes a full page reload with no loading indicator), and the match list has no visual hierarchy — every game looks equal. This plan adds both.

## Output
- `src/app/(app)/matches/sync-button.tsx` — `'use client'` component wrapping the submit button with `useFormStatus` loading state
- `src/app/(app)/matches/post-game-summary.tsx` — server component rendering the most recent match as a prominent summary card
- `src/app/(app)/matches/page.tsx` — updated to use SyncButton and PostGameSummary
</objective>

  <action>
    Create `src/app/(app)/matches/sync-button.tsx` as a `'use client'` component.

    Import `useFormStatus` from `'react-dom'`.

    The component renders a `<button type="submit">` inside the form (it must be a child of the form to receive status):
    - `useFormStatus()` returns `{ pending }` — a boolean that is true while the form submission is in flight
    - When pending: disable the button and show "Syncing…" text
    - When not pending: show "Sync" text
    - Keep the same className as the existing button in page.tsx

    The component signature: `export function SyncButton()`

    IMPORTANT: `useFormStatus` only works when the component is a *child* of a `<form>` element.
    The form tag stays in `page.tsx` (server component); only the button is extracted here.

    Avoid: wrapping the form itself in this component — the form must remain in page.tsx.
    Avoid: using useState + onClick fetch — the sync route returns a redirect (307),
    which doesn't work cleanly with fetch. useFormStatus is the correct approach here.
  </action>

    Props interface — accept the same shape as the match select in page.tsx:
    ```ts
    interface RecentMatch {
      id: string
      champion_name: string | null
      kills: number
      deaths: number
      assists: number
      cs: number
      win: boolean
      queue_type: string
      game_duration_seconds: number
      captured_at: string
    }

    export function PostGameSummary({ match }: { match: RecentMatch })
    ```

    Render a `<div>` card above the list with:
    - Label: "Last Game" (small, muted)
    - Champion name (large, bold)
    - Win/loss badge (blue for WIN, red for LOSS — same colour pattern as MatchCard)
    - K/D/A: "{kills}/{deaths}/{assists}"
    - KDA ratio: `deaths === 0 ? 'Perfect' : ((kills + assists) / deaths).toFixed(2)` + " KDA"
    - CS count
    - Duration formatted as mm:ss
    - Queue type with underscores replaced by spaces

    Use `rounded-lg border p-4` as outer container with a subtle left accent:
    - `border-l-4 border-l-blue-500` if win
    - `border-l-4 border-l-red-500` if loss

    Keep the formatDuration helper inline (copy from match-card.tsx — do not import from it
    since match-card is 'use client' and we want this to stay a server component).
