  <name>Task 1: Add GET /api/matches/[matchId] route</name>
  <files>src/app/api/matches/[matchId]/route.ts</files>
  <action>
    IMPORTANT: This project runs Next.js 16. Dynamic route params are
    `{ params: Promise<{ matchId: string }> }` — always await params.
    Follow the same import pattern as existing route handlers (Request, not NextRequest).

    Create GET handler that:
    1. Creates Supabase server client
    2. Calls supabase.auth.getUser() — returns 401 if no user
    3. Awaits params to get matchId (UUID)
    4. Queries `matches` table: `.from('matches').select('*').eq('id', matchId).maybeSingle()`
       RLS will filter to user's own matches — any other match returns null → 404
    5. Returns 404 if profile is null, 200 with `{ match }` if found

    Return type on success: `{ match: Tables<'matches'> }`
    Import `Tables` from `@/types/database`.

    Avoid: reading `raw_data` JSONB and re-parsing it in this route —
    return the full row as-is. The detail page can pick the columns it needs.
  </action>
  <verify>
    tsc --noEmit: no errors in this file.
    Manual: curl with valid auth cookie → 200; without → 401; unknown id → 404.
  </verify>
  <done>AC-1 satisfied: GET /api/matches/[matchId] returns correct status codes</done>
</task>
