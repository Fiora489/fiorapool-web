## Goal
Add "Load more" pagination to the match history list and apply mobile-first responsive polish across the matches pages.

## Purpose
`page.tsx` currently hard-caps at 20 matches with no way to see older games. `/api/matches` already accepts `start` + `count` params — the list just needs a client component to drive pagination state. Mobile polish (responsive padding, touch-friendly spacing) completes the Phase 2 ROADMAP scope ("mobile-first match list").

- `src/app/(app)/matches/[matchId]/page.tsx` — responsive padding + layout pass (no functional change)

  <name>Task 1: MatchList client component with load-more pagination</name>
  <files>src/app/(app)/matches/match-list.tsx</files>
  <action>
    Create `src/app/(app)/matches/match-list.tsx` as a `'use client'` component.

    **Type — reuse the same shape as MatchCard's Match interface:**
    ```ts
    interface Match {
      id: string
      game_id: number
      champion_name: string | null
      kills: number
      deaths: number
      assists: number
      cs: number
      win: boolean
      queue_type: string
      game_duration_seconds: number
      captured_at: string
    }

    interface Props {
      initialMatches: Match[]
    }
    ```

    **State:**
    ```ts
    const [matches, setMatches] = useState<Match[]>(initialMatches)
    const [loading, setLoading] = useState(false)
    const [hasMore, setHasMore] = useState(initialMatches.length === 20)
    ```
    - `hasMore` starts true only when the initial batch is exactly 20 (could be more pages)
    - If initial batch < 20, there are no more pages — hide the button immediately

    **Load more handler:**
    ```ts
    async function loadMore() {
      setLoading(true)
      const res = await fetch(`/api/matches?start=${matches.length}&count=20`)
      const data = await res.json()
      const next: Match[] = data.matches ?? []
      setMatches(prev => [...prev, ...next])
      if (next.length < 20) setHasMore(false)
      setLoading(false)
    }
    ```

    **Render:**
    ```tsx
    return (
      <div className="space-y-4">
        <div className="space-y-2">
          {matches.map((m) => (
            <MatchCard key={m.id} match={m} />
          ))}
        </div>

        {hasMore && (
          <button
            onClick={loadMore}
            disabled={loading}
            className="w-full rounded-lg border border-border py-2 text-sm text-muted-foreground hover:bg-muted disabled:opacity-50 transition-colors"
          >
            {loading ? 'Loading…' : 'Load more'}
          </button>
        )}
      </div>
    )
    ```

    Import `MatchCard` from `'./match-card'`.
    Import `useState` from `'react'`.

    Do NOT make this component async — client components cannot be async.
  </action>

    AC-1, AC-2, AC-3 satisfied.
  </verify>
