<objective>
## Goal
Promote `/rivals` from a basic add/remove list to a real rival-tracking page: head-to-head comparison strip (your stats vs each rival's), per-rival enrichment (streak, top champion, top role), polished cards, and isolated client islands for mutations.

## Purpose
The current page shows winRate + KDA + recent form per rival but offers no comparison to your own stats. The whole point of "rivals" is comparison — make it explicit.
</objective>

<acceptance_criteria>

## AC-1: GET /api/rivals Enriched
```gherkin
Given the user has 1+ rival
When GET /api/rivals is called
Then each rival returns existing fields PLUS streak (signed integer; +N for win streak, -N for loss streak),
     topChampion ({ name, games } or null), topRole (string or null)
And the response also includes self: { winRate, avgKda, games, recentForm, streak, topChampion } from the user's recent matches
```

## AC-2: Page Renders Head-to-Head Strip
```gherkin
Given the user is on /rivals
When the page loads
Then a HeadToHeadStrip shows "You" stats next to each rival's stats with delta indicators (+ or - per metric)
```

## AC-3: Add/Remove Still Work
```gherkin
Given the user adds or removes a rival
When the form submits or remove button clicks
Then the list updates without a full page reload
```

## AC-4: Mobile Functional
```gherkin
Given the user visits /rivals on a 390px mobile viewport
When the page loads
Then layout has no horizontal overflow; head-to-head strip stacks vertically on mobile
```
